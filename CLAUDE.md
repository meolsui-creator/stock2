# CLAUDE.md

## 프로젝트 컨텍스트

종목명을 입력하면 애널리스트들이 협업해 리서치를 만들고 PPTX 리포트로
내보내는 주식 분석 프로젝트입니다.

- 사용자는 증권사 리서치 담당입니다.
- 분석은 **가치투자 관점**으로 수행합니다.

## 위임 규칙

요청 성격에 따라 아래 기준으로 담당 애널리스트에게 위임합니다.

| 요청 유형 | 담당 | 도구 | 의존성 |
|---|---|---|---|
| 뉴스·공시 이슈, 시장 심리(센티먼트) 분석 | `news-sentiment-analyst` | 웹서치(키 불필요) | 없음 (독립) |
| 재무(②) 분석 | `fundamental-analyst` | DART OpenAPI (`.env`의 `DART_KEY`) | 없음 (독립) |
| 차트(③) 분석 | `market-technical-analyst` | FinanceDataReader (키 불필요) | 없음 (독립) |
| 리스크(④) 분석 | `risk-manager-analyst` | pykrx (키 불필요) | **뉴스·②·③ 산출물 필요** |
| 종합의견(⑤) | 각 애널리스트 산출물을 취합해 직접 작성 | — | 모든 산출물 필요 |

위임 원칙
- 각 애널리스트의 산출물은 **출처·기준일**을 반드시 포함해야 받아들입니다.
- 종합의견(⑤)은 위임하지 않고, 수집된 산출물을 근거로 직접 종합합니다.
- 새 애널리스트가 등록되면 위 표에 행을 추가해 위임 대상을 확장합니다.

## 오케스트레이션 (병렬은 상황에 맞게)

**무조건 병렬이 아니라, 요청 범위와 의존성을 보고 판단합니다.**

1. **단일 파트 요청** (예: "삼성전자 재무만 봐줘", "차트만") → 해당 애널리스트 **하나만** 위임합니다. 병렬화하지 않습니다.
2. **전체 리포트/복수 파트 요청** (예: "삼성전자 리포트 만들어줘") → 의존성에 따라 단계를 나눕니다.
   - **1단계(병렬 가능)**: 서로 독립인 뉴스·재무(②)·차트(③)는 **한 번에 동시 위임**합니다.
   - **2단계(직렬)**: 1단계 산출물이 모두 모인 뒤 `risk-manager-analyst`(④)에 전달합니다. 리스크는 앞 산출물에 **의존하므로 병렬하지 않습니다.**
   - **3단계(직접)**: 전체 산출물을 취합해 종합의견(⑤)을 직접 작성합니다.
3. **병렬을 피해야 하는 경우**: 산출물 간 의존이 있을 때, 사용자가 순차 진행을 원할 때, 앞 단계 결과를 보고 다음 위임 여부를 정해야 할 때는 **직렬로** 진행합니다.
4. 동시 위임 시에는 **한 메시지에서 여러 Agent 호출**을 함께 보내 실제로 병렬 실행되게 합니다.
5. 어떤 분석을 병렬로 돌리고 어떤 것을 직렬로 두는지, 진행 전에 **한 줄로 사용자에게 알립니다.**

## 산출물 규칙

- 리포트는 **5부 구성**입니다: ① 표지 ② 재무 ③ 차트 ④ 리스크 ⑤ 종합의견
- 모든 서술은 **"~입니다" 체**로 작성합니다.
- 모든 수치에는 **출처와 기준일**을 함께 적습니다.
- 취합한 리서치는 `reports/{종목명}.md`로 저장하고, PPTX 내보내기는 **`report-pptx` 스킬**을 사용합니다(출력: `reports/pptx/{종목명}.pptx`). 슬라이드 순서는 표지 → 종목 개요 → 재무 → 가격/추세 → 뉴스·심리 → 리스크 → 한 줄 종합으로 고정됩니다.

## 가드레일

- 매수·매도를 단정하는 표현은 금지합니다. **판단 근거 제시까지만** 합니다.
- **출처 없는 수치**는 사용하지 않습니다.
- 리포트 끝에 **학습용 분석임**을 명시합니다.
